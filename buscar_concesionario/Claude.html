¿De dónde salen los nombres en el autocompletado?

La lista predictiva no está embebida en el HTML de la página. Lo que pasa es lo siguiente:

Mecanismo típico: llamada AJAX a una API interna

Cuando empiezas a escribir en el campo "Nombre o denominación social", el formulario hace una petición HTTP en tiempo real (AJAX/fetch) a un endpoint del backend del RPC, algo como:

GET https://rpc.ift.org.mx/vrpc/api/concesionarios?q=TELECOMUNICACION

Ese endpoint consulta la base de datos del IFT y devuelve los nombres que coinciden con lo que escribiste. Es un autocomplete dinámico, no una lista precargada.

¿Puedes obtener TODOS los nombres?

Sí, pero no hay una ruta directa obvia. Hay dos caminos:

Opción A — Usar el buscador con campo vacío o búsqueda general Si dejas el campo en blanco y presionas "INICIAR BÚSQUEDA", el sistema devuelve resultados paginados con todos los concesionarios. Podrías scrapear esas páginas de resultados iterando la paginación.

Opción B — Interceptar la llamada de autocompletado Con las DevTools del navegador (F12 → pestaña Network), escribe una letra en el campo (por ejemplo "A") y observa qué petición HTTP se dispara. Verás la URL exacta del endpoint, sus parámetros, y el JSON que regresa. Probando con letras del abecedario ("A", "B", "C"...) podrías acumular todos los nombres existentes.

Cómo inspeccionar esto en tu navegador

  1. Abre https://rpc.ift.org.mx/vrpc/
  2. Presiona F12 → pestaña Network → filtra por XHR o Fetch
  3. Empieza a escribir en el campo de denominación social
  4. Observa las peticiones que aparecen — ahí verás la URL del endpoint y el JSON con los nombres

Cómo usarlo

1. Encuentra la URL del endpoint (el paso pendiente de DevTools):

  • Abre la página, F12 → Network → Fetch/XHR
  • Escribe "TEL" en el campo
  • Copia la URL de la petición que aparece

2. Pégala en el script, en la variable AUTOCOMPLETE_URL. Por ejemplo:

python
AUTOCOMPLETE_URL = "https://rpc.ift.org.mx/vrpc/api/concesionarios?q={query}"

El {query} es el marcador que el programa reemplaza automáticamente por cada prefijo.

3. Pon tus nombres en la lista NOMBRES_BUSCAR.

4. Ejecuta python buscar_concesionario.py

Qué hace el programa

  1. Descarga el catálogo completo consultando el endpoint letra por letra (A, B, C... + combinaciones) y deduplica los resultados. Lo guarda en catalogo_rpc.txt para que no lo tengas que descargar de nuevo.
  2. Hace fuzzy matching con un score que combina similitud de secuencia de caracteres + coincidencia de palabras clave, normalizado (sin acentos, mayúsculas, caracteres especiales).
  3. Muestra los 5 mejores candidatos con una barra de porcentaje:
    • > 70% → casi seguro que es el correcto
    • 40–70% → revisar manualmente
    • < 40% → probablemente no está en el catálogo

Cuando tengas la URL del endpoint compártela aquí y puedo ajustar el script si la estructura del JSON es diferente a lo esperado.
